性能的定义

计算机性能的定义需要考虑其用途（如：个人计算机对降低响应时间（response time）感兴趣，数据中心对吞吐量（throughput）感兴趣）
不同的应用关注计算机性能的不同方面。要改进一个程序的性能，必须明确性能定义，然后通过测量程序的执行时间来寻找可能的性能瓶颈
响应时间（response time）：也成为执行时间（execution time）或者消逝时间（elapsed time）或者墙上时钟时间（wall clock time），计算机完成某任务所需的总时间，包括硬盘访问、内存访问、I/O活动、操作系统开销和CPU执行时间等
吞吐率（throughput）：也称为带宽（bandwidth），性能的另一种度量参数，表示单位时间内完成的任务数量
通常情况下，响应时间和吞吐率相互影响

针对响应时间来讨论性能

性能 = 1/执行时间
性能比较

  • 性能X > 性能Y，则执行时间X < 执行时间Y
  • 性能X / 性能Y = 执行时间Y / 执行时间X = n

性能的度量

CPU执行时间（CPU时间）：只表示在CPU上花费的时间，而不包括等待I/O或运行其他程序的时间

用户感受到的是程序的响应时间，而不是CPU执行时间

用户CPU时间：程序本身消耗的时间
系统CPU时间：为执行程序而花费在操作系统上的时间

通常精确区分系统CPU时间和用户CPU时间是很困难的，因为难以分清哪些操作系统的活动属于哪个用户程序，而且不同操作系统的功能也千差万别

系统性能：表示空载系统的响应时间
CPU性能：表示用户CPU时间

作为计算机用户，关心的是响应时间。但是作为计算机设计者，需要考虑如何度量计算机硬件完成基本功能的速度，几乎所有的计算机都用时钟来驱动硬件中发生的各种事情
时钟周期（clock cycle）：时钟间隔的时间，也称为tick、clock tick、clock period、clock或者cycle
时钟频率（clock rate）：时钟周期的倒数
时钟长度：每个时钟周期持续的时间长度

指令的性能

计算机是通过执行指令来运行程序的，因此执行时间一定依赖于程序中的指令数

CPU时钟周期数 = 程序的指令数 * 每条指令的平均周期数
CPI：每条指令的时钟周期数，表示执行某个程序或程序片段时每条指令所需的时钟周期平均数

不同的指令需要的时间不同，CPI是一个程序全部指令所用时钟周期数的平均值

CPU性能

CPU性能公式

用户和设计者以不同的关注点看待性能，用户关心响应时间，设计者关心时钟周期。确定两者之间的关系，就能确定一个设计的变化对性能的影响
程序的CPU执行时间 = 程序的CPU时钟周期数 * 时钟周期时间
程序的CPU执行时间 = 程序的CPU时钟周期数 / 时钟频率

减少时钟周期数的同时可能引起时钟周期时间的增加，所以需要在亮哥因素之间进行权衡

指令数：执行某程序所需的总指令数量
程序的CPU执行时间 = 指令数 * CPI * 时钟周期时间
程序的CPU执行时间 = 指令数 * CPI / 时钟频率

如何确定CPU性能公式中因素的值

时钟周期时间

计算机说明书中有时钟周期时间

时钟周期时间

通过运行程序来测量CPU的执行时间

指令数

  • 通过用体系结构仿真器等软件工具预执行程序来测量出指令数
  • 通过大多数处理器中的硬件计数器来测量执行的指令数、平均CPI、性能损失源等

指令数取决于计算机体系结构，并不依赖于计算机的具体实现，因而可以在不知道计算机全部实现细节个情况下对指令数进行测量

指令组合（instruction mix）：在一个或多个程序中，指令的动态使用频度的评价指标

CPI

CPI与计算机的各种设计细节密切相关，包含存储系统和处理器结构，以及应用程序中不同类型的指令所占的比例，因此，CPI对于不同应用程序是不同的，对于相同指令集的不同实现方式也是不同的

  • 通过大多数处理器中的硬件计数器来测量CPI等

计算机的性能

比较两台计算机时，必须考虑指令数、CPI、时钟周期（或者时钟频率），三者组合才能确定执行时间

程序性能

硬件或软件指标 影响什么 如何影响
算法 指令数，CPI 算法决定源程序执行指令的数目，从而CPU执行指令的数目，可能通过较快或者较慢的指令影响CPI（例如：当算法使用更多的除法运算时，将会导致CPI增大）
编程语言 指令数，CPI 编程语言中的语句必须翻译为指令，从而决定了指令数。也可能影响CPI（例如：Java语言充分支持数据抽象，因此将进行间接调用，需要使用较高的CPI指令）
编译程序 指令数，CPI 编译程序决定了源程序到计算机指令的翻译过程，所以编译程序的效率即影响指令数又影响CPI，编译器会以复杂的方式影响CPI
指令集体系结构 指令数，CPI，时钟频率 指令集体系结构影响CPU性能的所有三个方面，完成某功能所需的指令数、每条指令的周期数以及处理器的时钟频率

功耗墙

时钟频率和功耗的增长是正相关

占统治地位的集成电路技术是CMO（互补型金属氧化半导体），其主要的能耗来源是动态能耗，即晶体管开关过程中产生的能耗（晶体管从0翻转到1或从1翻转到0消耗的能量）
动态能耗取决于每个晶体管的负载电容和工作电压
0→1→0或者1→0→1：能耗 ∝ 负载电容 * 电压^2
0→1或者1→0：能耗 ∝ 1/2 * 负载电容 * 电压^2
0→1或者1→0：功耗 ∝ 1/2 * 负载电容 * 电压^2 * 开关频率
开关频率：开关频率是时钟频率的函数
负载电容：链接到输出上的晶体管数量（称为扇出）和工艺的函数，该函数决定了导线和晶体管的电容

如何减少功耗

每次工艺更新换代，都通过降低电压来大幅减少能耗和功耗。一般来说，每代的电压降低大约15%，所以时钟频率增长为1000倍，但是功耗只增长了30倍
时钟频率的增加，导致功耗的增加，并且功耗已经到达了极限，无法再将处理器冷却下来

多处理器

为什么会出现多处理器

功耗的极限迫使微处理器的设计产生了巨变：单片微处理器中加入多处理器，以求更大的吞吐率，不再追求降低单个程序运行在单个处理器上的响应时间
processor：处理器
micprocessor：微处理器或者多核处理器

多处理器如何改善性能

程序员编写程序时充分利用多处理器的优势，可以显著改进响应时间

为什么编写显式并行程序如此困难

  • 并行编程必须准确、必须能够解决重要问题、必须为用户或其他程序提供接口使用
  • 发挥并行硬件的速度，必须将应用划分相同数量的任务到每个核，并同时完成；必须减少调度开销
